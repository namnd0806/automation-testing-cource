# 🎯 PHẦN 25: BEST PRACTICES & CAREER

> **Mục tiêu**: Best practices và chuẩn bị cho career path.

## Best Practices

1. **Naming**: Clear, descriptive names
2. **DRY**: Don't Repeat Yourself
3. **Wait**: Dùng Explicit waits
4. **POM**: Follow Page Object Model
5. **Data-Driven**: Tách data ra khỏi code
6. **Reporting**: Generate reports
7. **Version Control**: Use Git

## Career Path

- **Junior**: 0-2 years, Viết tests, maintain framework
- **Mid**: 2-4 years, Design framework, mentor juniors
- **Senior/SDET**: 4+ years, Architecture, CI/CD, tools

## Interview Prep

- Locator strategies
- POM pattern
- TestNG annotations
- Waits (Implicit vs Explicit)
- Handling alerts, frames, windows
- Data-driven testing
- CI/CD integration
